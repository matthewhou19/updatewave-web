## 1. 背景与定位升级

### 当前状态
UpdateWave Web 在 Vercel 上线,通过 Stripe 收取 `$25/reveal` 让 GC 解锁建筑师联系方式。

### 定位升级

| 维度 | 旧模型 | 新模型 |
|---|---|---|
| 定位 | Lead marketplace | Relationship intelligence |
| 主力产品 | 单条高时效 lead | 城市年度 hot architect list |
| 叙事 | "这里有新机会,快抢" | "掌握建筑师关系 = 业务安全感" |
| 对标 | HomeAdvisor / BidClerk | BuildingConnected / Dodge Data |

### 核心洞察

- **Lead = 战术**,高压、易错、失败成本高
- **List = 战略资产**,可复利经营的关系网
- 成功转化 1 个高产建筑师 >> 追几个热点 lead
- "Hot" 不给硬数字(避免对手逆向 + 避免上榜者维权),以定性叙述 + 过往项目举例呈现

## 2. 定价结构

| 产品 | 价格 | 定位 |
|---|---|---|
| 单条 lead | $129 | **Decoy** / 惩罚性单买价 |
| 城市年度 hot list | $499 × 3 城市 | **主力产品** (relationship asset) |
| 城市/区域 research | $1999 | **旗舰产品** (spec 未定) |

### Decoy 机制

- $499 ÷ 30 架构师 ≈ $16.63 / 架构师
- 显著低于 $129 / 单条
- 驱动客户选择年度订阅,而非单条消费

### 当前数据点 (N=1)

先前以 $100/条 卖给 1 个客户,他 7 选 5 = $500 营收。验证了 transactional 侧有需求,但 $499 年度订阅侧尚未验证 — ship-and-see 是本周的验证方式。

### 4.2 数据模型

**新表 `city_lists`:**
```sql
CREATE TABLE city_lists (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  city TEXT NOT NULL,                -- 'sf', 'sj', 'fremont'
  year INTEGER NOT NULL,             -- 2025
  title TEXT NOT NULL,
  description TEXT,
  price_cents INTEGER NOT NULL,      -- 49900
  architect_count INTEGER NOT NULL,  -- 30
  pdf_storage_path TEXT NOT NULL,    -- 'sf-2025.pdf'
  sample_architects JSONB,           -- 3 个预览架构师
  active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(city, year)
);
```

**新表 `list_purchases`:**
```sql
CREATE TABLE list_purchases (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id),
  city_list_id UUID NOT NULL REFERENCES city_lists(id),
  stripe_session_id TEXT NOT NULL UNIQUE,  -- 幂等保证
  purchased_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, city_list_id)
);
```

**Supabase Storage Bucket `city-lists-pdfs`:**
- Private bucket
- 访问走 30 天签名 URL
- 文件命名: `{city}-{year}.pdf`

**RLS 策略:**
- `city_lists` anon 读: `id`, `city`, `year`, `title`, `description`, `price_cents`, `architect_count`, `sample_architects`(用于营销页)
- `city_lists.pdf_storage_path` 仅 service_role 可读
- `list_purchases` 仅 service_role 可读写

### 4.3 路由设计

| 路径 | 类型 | 访问权限 | 功能 |
|---|---|---|---|
| `/list/[hash]/sf` | Server Component | 需要 hash | SF 城市营销页 + CTA |
| `/list/[hash]/sj` | Server Component | 需要 hash | San Jose 营销页 |
| `/list/[hash]/fremont` | Server Component | 需要 hash | Fremont 营销页 |
| `/list/[hash]/[city]/success` | Server Component | 需要 hash + 已购买 | 购买确认 + 下载按钮 |
| `/api/create-list-checkout` | Route Handler | POST | 创建 Stripe Checkout |
| `/api/download-list/[hash]/[city]` | Route Handler | GET | 返回签名 URL |

### 4.4 营销页组件 (`/list/[hash]/[city]`)

1. **Hero:** 城市 + 年份 + "Top N Hot Architects 2025"
2. **Why this matters:** 3-4 段,强调 relationship asset > lead
3. **Preview:** 3 个 sample 架构师(完整信息 + 2 个过往项目)
4. **"And 27 more":** 折叠块,显数量不显内容
5. **Price Comparison Component:**
   - `$499 — 整份 (30 位架构师,$16.63/位)`
   - `$129 — 单条 lead`
   - 突出 list 的单位性价比
6. **CTA:** 主按钮 "Buy SF List — $499"
7. **FAQ:** 3-5 条(交付形式 / 更新频率 / 退款政策)
8. **Cross-sell Footer:** 链接到另两个城市列表

### 4.5 Webhook 分支逻辑

现有 webhook 只处理 `reveals`,扩展为:

```ts
const productType = session.metadata?.product_type

if (productType === 'list') {
  await handleListPurchase(session)  // 新逻辑
} else {
  await handleRevealPurchase(session)  // 现有逻辑
}
```

`handleListPurchase`:
1. 读 `metadata.city_list_id`, `metadata.user_id`
2. INSERT INTO `list_purchases`(幂等由 UNIQUE 约束保证)
3. 返回 200

### 4.6 下载 API

`/api/download-list/[hash]/[city]`:
1. 从 hash 解出 `user_id`
2. 查 `list_purchases` 是否存在 `(user_id, city_list_id)`
3. `supabase.storage.from('city-lists-pdfs').createSignedUrl(path, 7200)`(2 小时有效)
4. 返回 `{ url: signedUrl }`
5. 前端 `window.location = url` 触发下载

### 4.7 E2E 测试覆盖

| 场景 | 断言 |
|---|---|
| 无 hash 访问营销页 | 404 |
| 有效 hash 访问营销页 | 渲染价格 + 3 sample |
| 点击 Buy → Stripe Checkout | 跳转 Stripe |
| Webhook 幂等(同 session 两次) | 只插入 1 行 |
| 购后访问 success 页 | 显示下载按钮 |
| 未购访问 success 页 | 跳回营销页 |
| 下载 API 对未购者 | 403 |
| 下载 API 对已购者 | 返回签名 URL |

### 4.8 首单监控指标

- 首单用时(从 banner 曝光到成交)
- 营销页 → Checkout 转化率
- Checkout → 付款成功率
- 付款 → 下载按钮点击率
- `/list/[hash]/sf` vs `/list/[hash]/sj` vs `/list/[hash]/fremont` 城市分布

## 5. Tier 2 WS 占位(spec 前置约束)

### WS-3: $1999 City / Region Research

**已知约束:**
- 必须明显区别于 $499 list(否则没人付 4× 价)
- 候选差异化维度: 跨建筑师分析 / 历史趋势 / 市场预测 / 特定类型深挖 / 竞争对手映射

**待决:**
- 报告内容结构
- 一次性 vs 订阅
- 交付形式(PDF / Deck / Live 演示)
- 目标买家(GC 领导层 vs 运营 vs 战略)

**Spec 时机:** WS-1 上线 2 周后,基于 $499 买家反馈

---

### WS-4: 客户教育文章

**已知约束:**
- 目的: 让 $499 买家真正用上 list,提升留存/复购
- 内容方向: 如何做 outreach / 建关系 / 跟进
- 分发: PDF 内链接 + 购后邮件(未来) + `/resources` 页

**待决:**
- 文章数量与深度
- 写作资源(内写 / 外包 / AI 辅助)
- 发布节奏

**Spec 时机:** WS-1 首批客户反馈后

---

### WS-5: Hash → 轻量账号

**已知约束:**
- 当前 hash 模型在 $499 场景的可能问题: 丢链接 / 想分享 / 要公司收据
- 候选方案: email + magic link(Supabase Auth 现成)
- 不能破坏现有 `/browse/[hash]` `/reveals/[hash]` 流程

**待决:**
- 是否值得迁移(数据说话)
- 混合模式(新功能 email,老功能 hash)
- 迁移路径(双跑 vs 一次切换)

**Spec 时机:** WS-1 上线 2 周后,基于真实摩擦点数据

## 7. 风险与对策

| 风险 | 对策 |
|---|---|
| $499 转化不及预期 | 上线 2 周内根据数据调整定价 / 内容 / 文案 |
| 买家丢失下载链接 | hash URL 永久有效,随时可重访 success 页 |
| "Hot" 定义被质疑 | 坚持定性叙述,不公布算法 |
| 现有 $25 客户流失 | 接受(用户决策),$499 新客户 LTV 更高 |
| 设计师(PDF / 网页)延期 | 7 天内留 1 天 buffer,必要时简化 v1 视觉 |
| 邮件送达问题 | v1 不依赖邮件,Stripe 自带收据 + hash URL 永久有效 |
| Webhook 重复触发 | UNIQUE(user_id, city_list_id) 约束幂等保证 |
